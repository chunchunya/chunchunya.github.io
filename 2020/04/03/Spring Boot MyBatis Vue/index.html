1 项目初始化

1.1 新建项目

​ 利用 idea 创建一个新的项目（也可使用前往( Spring Boot Starter) 制作项目包然后用IDE打开）

1.2 选择Spring Initializer

1.3 设置名称

设置Group、Artifact、Package等名称

添加Web下的Spring Web Starter、SQL下的MySQL Driver，MyBatis Framework

2 pom.xml 配置

2.1 Maven Repository简介

下载pom.xml中的依赖时可以在 Maven Repository中搜索并下载，以下以阿里巴巴的druid为示例

2.2 配置 MyBatis Generator 插件

在 plugins 中添加 plugin 配置，要注意其中的 mysql连接驱动 的版本，在6.0以上某些版本会出现问题。如果出现错误，复制错误提示Google一下。

其中的${basedir}是指groupid、artifactid、package等的组合，要根据自身情况而定，不复制整个pom.xml文件的话不会出现问题。在pom.xml文件中顶部包含有groupid、artifactid、package等。

2.3 数据库连接与MyBatis映射

3 数据库部分

3.1 新建数据库

​ 使用MySQL新建数据库，数据库名称自定义。

3.2 运行数据库脚本

​ 使用工具或命令行运行数据库脚本，创建示例数据库表、插入示例数据。

3.3 配置 generatorConfig.xml

在resource目录下新建generator目录，在其下再新建generatorConfig.xml

在pom.xml中的plugin配置中用到了此部分，配置项众多，有兴趣可阅读 MyBatis Generator配置项

以上配置中每个<table>对应数据库中一张表，亦可使用 通配符 % 来配置，将会映射数据库下所有表

3.4 生成代码

  • 进行以上配置后便可开始生成代码，具体操作如图

  • 查看构建结果，生成实体类与mapper类，其中的mapper相当于以往的DAO(Data Access Object)

3.5 Service 层

  1. 在model与mapper同级目录下创建 service 包
  2. 新建接口 SysUserService，其中代码可直接复制自动生成的 SysUserMapper 中的代码

接口中的方法定义不需要加 public 关键字

  1. 在 service 包下新建 impl 包，在impl包下新建 SysUserServiceImpl类，并实现SysUserService接口

注意类名前 加上 @Service注解， 在对SysUserMapper进行 @Autowired 会报错，不必在意，这是Spring提升了安全验证，不影响使用.

3.6 单元测试

​ 单元测试不是必须的，但是是推荐的，可以校验自己的方法是否正确，数据是否正确。

单元测试需要 加 @Test 注解

​ 在test包下进行单元测试，可双击方法名右键 run xxx运行，亦可点击方法名前的绿色小图标运行

3.7 Controller 层

​ 1. 在service同级目录下controller包，创建编写控制层 controller 代码，具体如图

注意添加注解 @RestController，功能相当于 @ResponseBody与@Controller，其中@ReponseBody标识后会把返回数据转换成JSON格式

在项目入口处添加 @MapperScan(“”)让Spring Boot 来自动扫描 Mapper 映射，也可在每一个Mapper类上加@Mapper注解

  1. 启动Spring Boot应用

    在 main方法中右键 run xxx，或者代码上方工具栏 三角形图标 运行

3.8 验证

此数据为使用 Chrome 插件 JSON Viewer后的数据，默认情况下是显示一串字符串，亦可采用Postman进行测试。

8 打包

8.1 jar包

  • 点击Maven中得到LifeCycle下的install进行打包，注意不是Plugins下的install

8.2 查看

  • 打包成功后jar包会出现在项目根目录下的target文件夹中

8.3 运行jar包

  • 进入到jar包根目录下，执行java -jar xxxxx即可运行
  • 也可以把jar包移动到方便的位置上，比如服务器，桌面…
  • hint: 在执行java -jar xxx命令时，可以在输入java -jar后输入jar包开头的几个字母，然后按tab键将会自动补全文件名
